## 一、产品概述

### 1.1 产品定位

一款面向个体商户和小型批发商的**通用进销存微信小程序**，帮助用户管理商品的进货、销售、库存，以及客户赊账对账。以"极致简单、扫码即用"为核心体验，在微信生态内完成日常经营数据管理。

### 1.2 产品愿景

让每一个小店老板、每一个批发档口，都能用手机管好自己的生意——不用电脑，不用培训，扫码就能上手。

### 1.3 核心价值主张

| 维度 | 价值 |
|------|------|
| 速度 | 扫码开单 3 秒完成，比手工记账快 10 倍 |
| 准确 | 库存自动扣减，杜绝卖超；赊账一目了然，杜绝坏账 |
| 省心 | 利润报表自动算，不用再拿计算器摁半天 |
| 随时 | 微信小程序即开即用，不占手机空间，换手机数据不丢 |

## 二、目标用户画像

### 用户 A：个体零售店主

| 属性 | 描述 |
|------|------|
| 身份 | 便利店、服装店、五金店、杂货铺老板 |
| 年龄 | 30-55 岁 |

| 痛点 | 手工记账混乱，库存对不上，不知道赚了多少 |

| 付费意愿 | 中等偏高，需免费试用建立信任 |

| 年龄 | 28-50 岁 |

**库存计算规则：**

> 当前库存 = SUM(入库数量) - SUM(出库数量) + SUM(退货回退数量)  
> 以 `stock_log` 表为唯一真相源（Single Source of Truth），而非依赖 `product.stock_qty` 字段。

### 5.2 技术选型

| 层级 | 选型 | 选型理由 |
|------|------|----------|
| 小程序端 | 微信原生 + WeUI | 体积最小、启动最快、不引入框架复杂度 |
| 管理员后台 | 内嵌 SPA（Vue3 + Element Plus） | Go 服务内嵌静态资源，单二进制分发，无需额外部署前端服务 |
| 后端语言 | Go (Gin 框架) | 单二进制部署、内存占用极低、2C4G 轻松扛数千并发 |
| 数据库 | MySQL 8.0 | 事务支持（进销存必须 ACID）、成熟稳定 |
| ORM | GORM | Go 生态最成熟 ORM，自动建表、参数化防注入 |
| 缓存 | Redis | 微信登录态缓存、管理员 Session 缓存 |
| 反向代理 | Nginx | HTTPS 终结、静态资源、限流 |
| HTTPS 证书 | Let's Encrypt + acme.sh | 免费自动续签 |
| 部署 | Docker Compose | 一键启动、环境隔离、易迁移 |

### 5.3 核心数据模型

```sql

```

**核心设计原则：**

> 库存以 `stock_log` 为唯一真相源（Single Source of Truth）。  
> 任一时刻库存 = SUM(in) − SUM(out) + SUM(refund_back)，不依赖 `product.stock_qty`。

## 六、非功能需求

### 6.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 首屏加载 | ≤ 2 秒 | 小程序冷启动到首页可交互 |
| 商品搜索 | ≤ 300ms | 本地 500 以内商品搜索 |
| 开单操作 | ≤ 500ms | 扫码→加入购物车→确认收款 |
| 列表分页 | 每页 20 条 | 避免单次加载过多数据 |
| 库存扣减 | 原子操作 | 数据库行级锁或乐观锁，绝不允许超卖 |

### 6.2 安全要求

| 要求 | 实现方式 |
|------|----------|
| 身份认证 | 微信登录 openid + 服务端 JWT Token |
| 数据传输 | 全量 HTTPS（Nginx SSL 终结） |
| 数据隔离 | 所有数据按 user_id 隔离，API 中间件强制校验 |
| 敏感信息 | 手机号脱敏展示（138****1234） |
| SQL 注入 | GORM 参数化查询 |
| 接口限流 | 同一用户同一接口 60 次/分钟 |

### 6.3 数据安全保障

| 要求 | 实现方式 |
|------|----------|
| 自动备份 | MySQL 每天凌晨 3 点自动备份到服务器本地 |
| 备份保留 | 保留最近 7 天 |
| 手动导出 | 用户可随时导出商品、销售、客户数据为 Excel |

### 6.4 用户体验要求

| 要求 | 说明 |
|------|------|
| 扫码优先 | 开单/进货/新增商品，扫码按钮放在最显眼位置 |
| 大字体大按钮 | 目标用户年龄偏大，字号 ≥ 14px，触控区域 ≥ 44px |
| 操作反馈 | 每次关键操作有明确 Toast 提示（"开单成功"、"入库完成"） |
| 错误友好 | 网络异常展示"网络开小差了，请重试"，不展示技术错误堆栈 |
| 离线提示 | 检测网络状态，断网时明确提示用户检查网络 |

### 6.5 运维要求

| 要求 | 说明 |
|------|------|
| 日志 | API 请求日志、错误日志落盘，按天切割，保留 30 天 |
| 监控 | 接入腾讯云云监控，CPU/内存/磁盘超 80% 告警 |
| 部署 | Docker Compose 一键部署，README 含完整步骤 |
