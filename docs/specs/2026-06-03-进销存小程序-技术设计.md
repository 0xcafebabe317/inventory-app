### 2.5 网络请求层

`utils/request.js` 统一封装 `wx.request`，所有页面通过它调用后端：

```javascript
// 签名
request(url, method, data) => Promise<response>

// 自动行为:
// 1. 自动携带 Authorization: Bearer {jwt}
// 2. 401 → 自动用 refresh_token 换新 token → 重试原请求
// 3. 403 code=EXPIRED → 跳转 lock 页
// 4. 网络异常 → wx.showToast("网络开小差了，请重试")
```

### 2.6 认证流程

```
用户打开小程序
      ↓
wx.login() → 后端 code2session → 获取 openid
      ↓
  ┌─ openid 已注册？
  ├─ 是 → JWT access 有效？→ 是 → 进入首页
  │              ↓否
  │         用 refresh_token 换新 access
  │              ↓成功 → 进入首页
  │              ↓失败 → 重新 login
  │
  └─ 否 → 新用户注册流程：
          1. 展示手机号授权按钮（open-type="getPhoneNumber"）
          2. 用户点击授权 → 后端绑定 phone + openid
          3. 创建账户 (trial_start_at=now)
          4. 签发 JWT → 进入首页（7天试用开始）
```

**JWT 参数：**
- Access Token：2 小时有效
- Refresh Token：7 天有效，存于本地 Storage
- 载荷：`{ user_id, openid, iat, exp }`

### 2.7 无商业内容约束

为以个人身份通过小程序审核，前端全局遵守：

| 约束 | 实现 |
|------|------|
| 不展示任何价格数字 | 联系管理员页只有微信二维码 + "如需开通权限，请联系管理员" |
| 不展示套餐/方案 | 无任何卡片/价格/折扣 |
| 不出现"付费""订阅""购买" | 所有文案使用中性措辞 |
| 锁定页不含商业话术 | "试用期已结束，请联系管理员继续使用" |
| 服务通知文案中性 | "您的试用期将于X天后结束" |

### 4.3 库存扣减（行级锁）

```go
// service/stock.go
func (s *StockService) DeductStock(db *gorm.DB, userID, productID int64, qty int) error {
    return db.Transaction(func(tx *gorm.DB) error {
        // 1. 行级锁读取当前库存
        var product model.Product
        if err := tx.Set("gorm:query_option", "FOR UPDATE").
            Where("id = ? AND user_id = ?", productID, userID).
            First(&product).Error; err != nil {
            return err
        }

        // 2. 校验库存充足
        if product.StockQty < qty {
            return ErrInsufficientStock
        }

        // 3. 写入库存流水
        log := model.StockLog{
            UserID:    userID,
            ProductID: productID,
            Type:      "out",
            Qty:       qty,
            BeforeQty: product.StockQty,
            AfterQty:  product.StockQty - qty,
            RefType:   "sale",
        }
        if err := tx.Create(&log).Error; err != nil {
            return err
        }

        // 4. 更新冗余库存字段
        return tx.Model(&product).
            Update("stock_qty", gorm.Expr("stock_qty - ?", qty)).Error
    })
}
```

### 4.5 错误码设计

| 错误码 | 含义 | 小程序端行为 |
|--------|------|-------------|
| `OK` | 成功 | - |
| `UNAUTHORIZED` | JWT 无效/过期 | 自动刷新或重新登录 |
| `EXPIRED` | 试用到期/订阅到期 | 跳转锁定页 |
| `FORBIDDEN` | 无权限 | Toast 提示 |
| `INSUFFICIENT_STOCK` | 库存不足 | Toast "库存不足，当前剩余X" |
| `NOT_FOUND` | 资源不存在 | Toast 提示 |
| `VALIDATION_ERROR` | 参数校验失败 | Toast 具体错误字段 |

| `INTERNAL_ERROR` | 服务端异常 | Toast "网络开小差了，请重试" |

### 5.2 库存一致性保证

```
库存计算规则：
  任意时刻库存 = SUM(stock_log WHERE type='in').qty
                − SUM(stock_log WHERE type='out').qty
                + SUM(stock_log WHERE type='refund_back').qty

product.stock_qty 是冗余字段，每次库存变动时同步更新。
可通过 SUM(stock_log) 与 product.stock_qty 交叉校验。
```

| product | (user_id, category) | 按分类筛选 |
| sale_order | user_id + created_at | 销售列表分页 |
| stock_log | (user_id, product_id) | 单品库存流水 |
| stock_log | (ref_type, ref_id) | 按关联单据查流水 |
| customer | user_id | 用户自己的客户列表 |

## 八、安全设计

| 层级 | 措施 |
|------|------|
| 传输 | 全量 HTTPS，Nginx SSL 终结 |
| 认证 | 用户 JWT (2h) + 管理员 JWT (24h)，密钥环境变量注入 |
| 数据隔离 | 所有 SQL 查询带 `user_id` 条件，由中间件注入上下文 |
| 防注入 | GORM 参数化查询 |
| 限流 | 同用户同接口 60次/分钟 (Redis 计数器) |
| 密码 | 管理员密码 bcrypt 哈希存储 |
| 数据库 | 仅监听 127.0.0.1:3306，不对外暴露 |
| 备份 | 每日凌晨3点 mysqldump，保留7天 |
| 日志 | API 请求日志落盘，按天切割，保留30天 |
